Đây là tài liệu kỹ thuật đầy đủ bao gồm kiến trúc, bảo mật, và tối ưu hiệu năng. Developer cần implement toàn bộ theo đúng spec, không được bỏ qua bất kỳ mục bảo mật nào. Ưu tiên thực hiện theo thứ tự: Schema → Auth → Task/Callback → Wallet → Withdrawal → Admin.

## 🧭 Tổng quan dự án
Xây dựng một web platform vượt link kiếm tiền (link bypass earning platform) với các chức năng:
- User đăng ký/đăng nhập → vào `/task` nhận nhiệm vụ
- Mỗi nhiệm vụ là 1 link rút gọn từ provider bên thứ 3 (web1s, v.v.) — user vượt xong thì được cộng tiền vào ví
- User vào `/payment` để rút tiền về ngân hàng / Momo / crypto
- Admin quản lý toàn bộ qua dashboard + nhận thông báo qua Zalo/Telegram Bot

**Tech stack bắt buộc:**
- Backend: Go (Gin hoặc Fiber framework)
- Frontend: Next.js 14+ (App Router)
- Database: PostgreSQL 16
- Cache / Queue: Redis (dùng Upstash Redis free tier)
- Containerization: Docker + Docker Compose
- Deploy: Go API → Render.com, Next.js → Vercel (free)
- CI/CD: GitHub Actions (optional nhưng nên có)

### 1. Bypass Token — Chống giả mạo callback
**Vấn đề:** User có thể copy URL callback tự gọi mà không cần vượt link thật sự.
**Yêu cầu triển khai:**

// Tạo token có HMAC signature khi claim task
func GenerateBypassToken(claimID, userID string, secret string) string {
    payload := fmt.Sprintf("%s:%s:%d", claimID, userID, time.Now().UnixNano())
    mac := hmac.New(sha256.New, []byte(secret))
    mac.Write([]byte(payload))
    sig := base64.RawURLEncoding.EncodeToString(mac.Sum(nil))
    return fmt.Sprintf("%s.%s", base64.RawURLEncoding.EncodeToString([]byte(payload)), sig)
}
// Token có TTL 20 phút, lưu expiry vào DB (bypass_token_expires)
// Khi callback về: verify signature + check expiry + check status = 'pending'

3. Withdrawal — Atomic Balance LockVấn đề: User submit nhiều request rút cùng lúc, vượt số dư.Yêu cầu triển khai:SQL-- Dùng 1 query duy nhất, atomic check-and-lock:
UPDATE wallets
SET
    balance       = balance - $1,
    locked_amount = locked_amount + $1,
    updated_at    = NOW()
WHERE
    user_id = $2
    AND (balance - locked_amount) >= $1   -- Chỉ trừ tiền available
    AND status != 'banned'
RETURNING id;
-- Nếu không có row nào trả về → insufficient balance hoặc concurrent request

Rate Limiting (Redis)Triển khai middleware rate limit cho tất cả các endpoint:POST /auth/register: 5 lần / IP / giờPOST /auth/login: 10 lần / IP / 15 phútPOST /tasks/:id/claim: 20 lần / user / giờPOST /withdrawals: 3 lần / user / ngàyGET /callback/bypass: 5 lần / token (sau đó block)Go// Dùng sliding window counter trên Redis
func RateLimit(key string, limit int, window time.Duration) bool {
    pipe := redis.Pipeline()
    now := time.Now().UnixMilli()
    windowStart := now - window.Milliseconds()
    
    pipe.ZRemRangeByScore(ctx, key, "0", strconv.FormatInt(windowStart, 10))
    pipe.ZCard(ctx, key)
    pipe.ZAdd(ctx, key, redis.Z{Score: float64(now), Member: now})
    pipe.Expire(ctx, key, window)
    
    results, _ := pipe.Exec(ctx)
    count := results[1].(*redis.IntCmd).Val()
    return count < int64(limit)
}
